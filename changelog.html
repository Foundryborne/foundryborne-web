v1.6.1 Busy Capybara #

Foundryborne Release Announcement

The Foundryborne team is proud to announce the release of version 1.6.0 for the Foundryborne Daggerheart System.

Changelog

Features

  • Death Rolls: Fully implemented Death Rolls with dice rolling, effects, healing and scars.
  • Defeated Handling: Automation now toggles on/off the various conditions that occur during death moves.
  • Fate Rolls: Added chat commands for Fate Rolls.
  • Character Reset: Added the ability to reset a character through the character sheet context menu.
  • Rolltables: Enhanced the system's Rolltables so you can set multiple roll formulas and then select which one to use via a select.
  • Companion Levelup: Characters can now set an ActiveEffect to give their companion bonus levelup choices.
  • Companion Token: Characters with a companion can now rightclick their token to bring out and hide it away with clowncar behavior like how the party works.
  • Damage Multipliers: ActiveEffects can now set a damage multiplier -for- an actor, so they for instance deal double damage.
  • Incoming Damage Multipliers: ActiveEffects can now set a damage multiplier -on- an actor, so they for instance take double damage.
  • Scene Environments: SceneEnvironments will now register any triggers they have if their scene is active.
  • Fear/Hope Automation: /dr chat commands can now specify grantResources=true to make the roll use fear/hope automation.

Bugfixes

  • Resolved an issue where a multiclass character's spellcast modifier wouldn't be properly shown and used if the multiclass was a non-spellcaster and caster.
  • Resolved an issue where secret blocks in descriptions were not being shown because of a faulty call checking for ownership.
  • Resolved an issue where actions without descriptions had stopped their fallback behavior to show the Feature's description.
  • Resolved an issue where Companions' attacks did not add the character's instinct modifier correctly.
  • Resolved an issue where Companions' attack range ended up weird when it was increased by levelup.
  • Resolved an issue where using a Companion's Action button in the sheet to make a roll included the Character's experiences instead of the Companion's.
  • Improved performance when dragging many adversaries at the same time.

Compendium Updates

  • Updated Beastbound features that should give the companion bonus levelup choices to have effects that do that.
  • Seaborne community feature Know The Tide now has a trigger that automatically gives it a token when you roll with fear.
  • Updated Adversaries with Features using the new damage multiplier ActiveEffects.
  • Updated Rolltables to have multiple formulas for Common|Uncommon|Rare|Legendary.
  • Removed the trigger form the domain card Ferocity as it was triggering incorrectly. It'll be redone in a later version.

Version 1.6.1 #

Features

  • Added Automation system settings to disable automation of Death Moves.

Bugfixes

  • Resolved an issue where the RollDialog advantage section was not being shown when rolling.
  • Resolved an issue with LightMode chat, tokenHUD, and tooltips styling.
  • Resolved an issue where the /dr command did not default to granting resources if fear/hope automation is on.
  • Resolved an issue where the /fr, /dr and their button equivalents were not parsing parameters correctly.

Compendium Updates

v1.5.5 – Creative Capybara #

Foundryborne Release Announcement

The Foundryborne team is proud to announce the release of version 1.5.0 (Creative Capybara) for the Foundryborne Daggerheart System.

Changelog

Features

  • Roll Effect Toggles: In Roll Dialogs you can now toggle relevant effects that modify the outcome instead of having to active/deactive effects elsewhere.
  • Scene Environments: Added Scene Environments, allowing you to link environments to a scene for quick access.
  • Summon Action: Implemented the Summon action to allow easily summoning multiple adversaries via an action.
  • Trigger System: Added a Trigger system. This allows actions to register macros to run at specified occurrences in the system.
  • Item Descriptions: Weapons and Armors now display the descriptions of any Item Features they have in the item description.
  • Active Effects: Added system.rules.dualityRoll.defaultHopeDice and system.rules.dualityRoll.defaultFearDice as ActiveEffect paths to set the default hope/fear dice in the roll dialog for a character.
  • Domain Touched: Domain Cards can now setup Domain Touched, meaning they are disabled unless X amount of cards from their domain are in the characters loadout.
  • Vault Active: Domain Cards can now be set to Vault Active, meaning that their effects function even when they're in the vault.
  • Loadout Ignore: Domain Cards can now be set to Loadout Ignore, meaning that they don't take up any space in the loadout.
  • Beastform Scale: Beastforms can now set a Scale for the transform image if required.

Bugfixes

  • Resolved an issue where the light/dark mode could appear incorrectly in the Daggerheart sidebar menu.
  • Resolved an issue where Adversary custom Critical Threshold was incorrectly applying to reaction rolls.

Compendium Updates

  • Added Triggers for Strange Patterns and Ferocity.
  • Added Summon actions on relevant Adversaries.
  • Domain Cards updated with Domain Touched, Vault Active and Loadout Ignore.
  • Updated Adversaries and Environments with defaultHopeDice and defaultFearDice ActiveEffects to apply on characters.

v1.4.6 – Festive Capybara #

Foundryborne Release Announcement

The Foundryborne team is proud to announce the release of version 1.4.0 (Festive Capybara) for the Foundryborne Daggerheart System.

Changelog

Features

  • Adversary Token Sizes: Added TokenSize for Adversaries. These are size categories you can easily apply to adversaries to set them to predefined, uniform sizes (Tiny, Small, Medium, Large etc). You can edit the token sizes attached to them in the Homebrew Settings.
  • Adversary Features: Features for Adversaries/Environments now have an ActionForm (Passive|Action|Reaction) that is displayed next to their name in the sheet.
  • Tooltips: Large visual improvements on tooltips.
  • Sheet Improvements: Improved namefields in the sheets to wrap for longer names.
  • Chat Messages: Saving Throw entries in chat messages now display the token name rather than the actor name.
  • Currency Transfer: You can now drag and drop currency from one character/party to another to transfer the same way you do for quantities of items.

Bugfixes

  • Resolved an issue where players rolling reaction rolls from chat messages wouldn't register correctly.
  • Resolved an issue where rolling reaction rolls from chat messages could fail to work with DiceSoNice active.
  • Resolved an issue where using an experience and simultaneously receiving automated hope on rolling didn't work correctly.
  • Resolved an issue during levelup where you couldn't select an increase on an experience you gained during the levelup process.
  • Resolved an issue where subclass features a character didn't have access to would show up as about to be refreshed when taking Downtime.
  • Resolved an issue where the Extra-Damage effect from the BattlePoints section in the CombatTracker was being applied to Characters added to combat.
  • Resolved an error that could cause TagTeamRolls not to be created properly.
  • Resolved an issue with RangeDependant effect automation (such as the Paired item feature) to be based on player targeting something in range.
  • Resolved an issue where drag-selecting in inputs - such as resource inputs - on the character sheet lead to unexpected behavior.
  • Resolved an issue where chat styles broke in stream mode.
  • Resolved an issue where the Sheet Experience section would display incorrectly when an experience name was really long.

Compendium Updates

  • All adversaries have had TokenSizes added.
  • All beastforms have had TokenSizes added.
  • All adversaries have had the hardcoded instances of their names removed from features and actions, replaced with a dynamic @lookup. This means if you change the name of the adversary, or drag the feature to another adversary, the name in the text will be updated.
  • Removed adversary action descriptions that just copied the feature description. This means there's only one place to edit instead of two when changing it.
  • Wings Of Light: Corrected to be self-applied.
  • Conjure Swarm: Now correctly lowers incoming damage severity.
  • Elundrian Chain Armor: Name corrected.

As a note - since Features now have an attached ActionForm (Passive, Action, Reaction) but they didn't previously - existing adversaries outside of the updated SRD Compendium will have defaulted to show Passive.

v1.3.2 – Tactical Capybara

Foundryborne Release Announcement #

The Foundryborne team is proud to announce the release of version 1.3.0 (Tactical Capybara) for the Foundryborne Daggerheart System.

Changelog

Features

  • Effects Display: Added an Effects Display. The effects on a character will now be shown in the top right next to the sidebar for easy removal of effects via right-clicking.
  • Battlepoints Display: Added a Battlepoints Display for the GM in the combat tab, giving an overview of the balance of an encounter, as well as the ability to apply encounter wide effects such as extra damage for 2 BP. Homebrew effect support coming later.
  • Inventory Improvements: Added drag/drop reordering of items in the Character and Party inventory.
  • Drag/Drop: Adjusted drag/drop of features between sheets to work without being in edit mode.
  • Biography: Improved the character sheet biography tab for better editing and a more natural scroll experience.
  • Module Support: Added CSS variables to the :root scope for use by modules.

Bugfixes

  • Resolved an issue where Active Effects giving bonuses to Action Rolls were not applying to Trait Rolls from the character sheet.
  • Resolved an issue where Attack Actions with a saving throw would fail to follow Fear/Hope automation.
  • Resolved an issue where transferring items between sheets caused duplication with non-quantity items.
  • Resolved an issue where using the sidebar GM menu to do a Long Rest refresh didn't also refresh Resources/Actions that refresh on Short Rest.
  • Resolved an issue where Actions on Homebrew Item Features and Downtime Moves couldn't properly add Effects.
  • Resolved an issue where the weapon attack tooltip in sheets was displaying incorrect damage.
  • Resolved an issue where transforming back from a Beastform would fail if the character's token dimensions were smaller than 1.
  • Resolved an issue where transforming back from a Beastform could become temporarily blocked when a scene was deleted.
  • Resolved an issue where Tag Team Rolls didn't include the critical damage bonus for every participant if a critical result was selected.
  • Resolved an issue where, for Tag Team Rolls, the hope cost wasn't applied to the initiating character when the roll was made with fear.
  • Resolved an issue where creating a new domain card on the Character Sheet would error if you lacked the Arcane domain.

A note for module makers. There's now a writeup on the system theming and style rules and how to take advantage of them: Theming and Style Rules

Version 1.3.1 #

Features

  • Currency Settings: All currency denominations can now be disabled in the homebrew section if no coin tracking at all is desired.

Bugfixes

  • Resolved an issue where the Downtime dialog would fail to open.
  • Resolved an issue where TagTeamDialog crit damage was not correctly calculated when a multi-dice formula was present (ex: 2d6).
  • Fixed various localization misses.

Compendium Updates

  • Updated text for Book Of Grynn and Splintering Strike with errata text.
  • Improved Small Dagger is now correctly set as a secondary weapon.

Version 1.3.2 #

Features

  • Massive Damage: Added the variant rule for massive damage to the system settings.
  • Currency Inputs: Currency inputs are now delta inputs. You can for example write -4 or +4 and alter the value by that, rather than the final value.
  • Currency Icons: The currency inputs can now optionally have a font-awesome icon next to them.
  • Spotlight Request Queue: Added a Spotlight Request Queue checkbox setting that defaults to not being on. We've found that letting players request the spotlight and having it get ordered from oldest to newest can often be a bad thing, as it is essentially recreating an initiative order. You can still enable it if you do want this.

Bugfixes

  • Resolved an issue where Countdown automated progression wasn't working when triggered by a player due to permissions.
  • (Battlepoints) Calculation correction when multiples of one category were in the encounter. (IE, 3x Standard --> 6 BP)
  • Resolved an issue where the /dr command has working automation when used with a character again.
  • Fixed certain cases of hope/fear automation not setting the resources correctly on reroll. All in all, if you had issues with hope/fear automation, it should be solved with this version.

Compendium Updates

  • Guardian's Iron Will: Feature corrected to let you use an extra armor slot against damage, rather than spending 1 stress.
  • Multiple Adversaries/Environments have been corrected to use direct damage for certain features/attacks.
  • Wings Of Light: Stress cost added.

Please also remember to create a backup of your world before installing the update, just in case.

v1.0.6 – Sleepy Capybara

Foundryborne Release Announcement #

The Foundryborne team is proud to announce the first stable release of the Daggerheart Game System for Foundry VTT — v1.0.0 is now live!

We deeply appreciate your patience and support throughout development. This milestone marks just the beginning — we’re excited to bring even more features and improvements in future updates.

Installation

Installing the Daggerheart system is now easier than ever. No developer experience is required!

You can install the system directly from Foundry’s package browser:
👉 https://foundryvtt.com/packages/daggerheart

Search for 🔍 Daggerheart in the system installation section of Foundry's admin interface.

Or use our manifest URL to install the latest version:
📦 https://github.com/Foundryborne/daggerheart/releases/download/1.0.0/system.json

Features

Here’s what’s included in version v1.0.0:

  • Support for the 2D12 Hope & Fear dice system
  • Complete SRD content as world compendium

Version 1.0.1 #

New Features & Improvements

  • The option to drag features between adversary sheets to quickly add a copy of an existing feature to a new adversary has been added.
  • You can now right-click features that restore "per session". In the future there will be a "new session" for the GM that will automate this.
  • Custom icon artwork for the combat wheelchair has been added. Special thanks to @CyrensMaps for creating them!
    • In addition, the credits in the Welcome journal have been adjusted
  • Updated the system background image to include the Foundryborne logo.

Bug Fixes

  • The Feature "Unconscious" spelling has been corrected.
  • Adversary Features should now correctly remove the associated Fear cost.
  • The Issue with armor features that prevented some drag&drop actions on armor items has been resolved.
  • Resolved a crash on experience selection.
  • Resolved a subclass error on multiclassing.
  • Resolved an issue where multiclasses incorrectly gained the hope feature for the class.
  • Resolved an issue where Class/Subclass features were not properly deleted on delevel.
  • Removed automatic deletion of features on delevel when not using levelup auto.
  • Custom (Homebrew) Domains now have their name displayed correctly and can be select in the multiclass selection screen.
  • A fix for bards rally dice showing in damage dialog when it should not.
  • Resolved an issue with the "reliable" weapon feature not working as intended.
  • Resolved an issue where Seraphs Prayer Dice did not increase with the spellcast modifier.
  • Resolved an issue where the Seaborne community option was not correctly displayed on character sheets.
  • Resolved a display issue with action buttons in chat messages.
  • Improved the display of reaction rolls in chat, to make it more clear that roll was reaction roll and not a regular duality dice roll.
  • Resolved an issue with beastform transformations breaking when using wildcard images for the character or beastform.
  • Resolved an issue where control buttons were not visible on the fear tracker in bar mode under certain conditions.
  • Resolved an issue where Sneak Attack would not be displayed as an option.
  • Resolved several issues with the adversaries.
  • Resolved an issue that broke compendium sorting when importing a whole compendium.
  • Resolved various spelling mistakes.
  • Resolved an issue with action buttons being vesible despite not having the permission to use them.
  • Resolved an issue with some item, feature, and action names being displayed incorrectly.
  • Improved the tooltips for traits in the character sheet to display the foll name of each trait in the tooltip.
  • Resolved an issue with damage formulas based on hope/fear results.

Please note that for some of the bugfixes to apply you would have to recreate the player/companion actor, drag the adversary out from the compendium again or replace any updated item/feature on the character sheet.

Version 1.0.2 #

Forge

Following communication with Forge, conflicts that have persisted since launch appear to have been resolved. The system should now download and update normally. We appreciate confirmation from users testing this update.

Bug Fixes

  • Disabled v13 waypoints from being drawn while combat is active.
  • Adversary basic attacks now display labels in the sidebar, including range and damage.
  • Selecting experience increases during level-up no longer causes crashes.
  • Fixed an issue where spending and gaining Hope simultaneously could result in incorrect totals (-1 or +1 instead of zero).
  • Added new system settings in Appearance Settings to allow Features and other Items to be initially expanded or collapsed.
  • Corrected effect calculations to properly use values from the Item model. This fixes issues such as the Barrier item previously adding the character's Tier to armor instead of the Item's value.
  • Critical damage calculations have been corrected. Previously, dice rolls were always maximized in addition to adding the dice maximum, resulting in double-counted damage.
  • Actor portraits are now top-focused, improving visual layout.
  • Removed redundant result text from reaction chat messages (e.g., “With Hope,” “With Fear”) to reduce confusion.
  • Corrected downtime refresh behavior to prevent erroneous refreshes of Session or Scene cooldowns.
  • Improved Character Creation layout: the top navigation bar has been removed, and Equipment is now the last tab, ensuring it is not overlooked while remaining optional.
  • Resolved an issue where activating or deactivating beastforms could move characters to unintended positions on the map.
  • Resolved duplication of scrolling text messages (e.g., Hope +1, Stress -1) in certain situations.
  • Companions can now level up independently and are no longer tied to the ranger's level-up.
  • Removed unintended icons in the Combat Tracker introduced in version 1.0.1, cleaning up the interface.
  • Domain cards now display their level in the loadout list.
  • Custom Downtime Moves descriptions are now correctly shown in tooltips.
  • Improved visibility and behavior of action buttons in chat messages.
  • Resolved an issue where ancestries were missing in the SRD Sidebar.
  • Item Attachments have been temporarily removed until the backend work to fully support them is complete.
  • Resolved an issue where the movement style icon was not being displayed correctly in some cases.

Please note that for some of the bugfixes to apply you would have to recreate the player/companion actor, drag the adversary out from the compendium again or replace any updated item/feature on the character sheet.
