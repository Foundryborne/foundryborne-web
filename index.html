This is Daggerheart for Foundry.

Welcome to the unofficial, community made Daggerheart system for FoundryVTT.
This system is designed to capture the full flavor of the 2d12 Hope & Fear dice mechanics, character builds, and collaborative storytelling magic.

About the System

The system includes complete character creation using card choices, Hope and Fear dice mechanics, team tags, group checks, and unique features tailored for the Daggerheart experience.

v1.2.1 ‚Äì Partying Capybara

Foundryborne Release Announcement

The Foundryborne team is proud to announce the release of version 1.2 (Partying Capybara) for the Foundryborne Daggerheart System.

Changelog

New Features

  • Party Actor: Create a new Actor for your party! Including stats overview, shared inventory and a "clown car" mode to easily move the whole party around!
  • Light mode support: In the previous versions there were a few issues with people using the light mode. It is now fully supported and everything is readable, no matter which theme!
  • Group Rolls: Full support for group roles, with a custom group roll dialog.
  • Sheet display options: If you prefer to have "pips" instead of actual numbers for HP, Stress and Armor Slots (Just like the paper sheets), you can now find an option for this in the system settings!
  • New countdown system: We completely rewrote the old countdown system, and the new one should work a lot better.
  • Attribution Feature: Added Attribution to system items so that the author and other information can be set.
  • Beastform Wildcards: Added Support for Wildcard tokens for beastforms.
  • Direct Damage: Direct Damage is now fully supported!
  • Auto-Expanding Roll Messages: Find a new setting in the system options to auto-expand dice rolls in chat messages!
  • Better Homebrew support: You can now add custom featurs to Weapons & Armor, as well as custom Adversary types!
  • Per Scene measurments: You can now override the Daggerheart measurements for every scene individually. Set what Melee|Close etc means on the scene, or disable the system ruler to use standard Grid units instead.
  • New Daggerheart Menu: It has a lot of GM tools! Expect more in future updates.
  • Simple Sheets: A new sheet type to view actors with limited observer permissions.
  • Compendium Browser Update: Custom compendiums are now included in the compendium browser (and are now shown during character creation as well)!
  • Better Character Creation: Background and Connection questions are now added to all classes and can be added during character creation.
  • Better Weapons: Weapons now can have a custom damage formula!
  • Level Up Choices: You can now review the choices you made while leveling up your character.
  • Companion Protection Initiative: Companions no longer show up in the combat tracker. They are now save. Please don't hurt them.
  • Spotlight Tracker Improvements: You can now see the order in which players requested the spotlight!

Bug Fixes

  • Updated all SRD entries to make sure they reflect the latest errata.
  • Several Bugfixes & fixed typos, including wrong dice or damage type.
  • Fixed the display of chat messages that are not supposed to be displayed, as they are redundant.
  • Resolved several issues where effects were not or incorrectly applied.
  • Updated the default view of the Loadout and Vault views to List View.
  • Resolved several issues with the character creation process, and updated the order in which the characters are created to reflect the core rules.
  • Resolved several issues with the sheet layouts so they should look better even with longer names/descriptions, which also helps the community translations.
  • Resolved an issue where consumable items were not actually consumed on use.
  • Resolved an issue where observer permissions would allow a player to use or edit items on other players sheets.
  • Resolved an issue with scrolling battle text to better handle situations where multiple changes happen at the same time.
  • Updated the Font styling to be more consistent, which makes it easier for custom modules to override the fonts.
  • Resolved an issue where the hope cost for experiences would not be correctly automated in the dice roll dialog.
  • Resolved an issue where the description for your experiences tooltip would not be displayed in the dice roll dialog when hovering over an experience.

Version 1.2.1

Bug Fixes

  • Resolved an issue that would cause data migrations from the old countdown system to the new one to fail in some cases. This should work without needing to go to a backup.
  • Resolved an issue where the Countdown UI failed to position correctly if the sidebar was initially expanded.
  • Resolved an issue where the Countdown UI wasn't considering Foundry's UI-Scale setting and could become badly positioned.
  • Resolved an issue where the Countdown UI would disappear if a user hit Escape.
  • Resolved an issue where the 'clowncar' feature of the Party token didn't work properly if you used it on an inactive scene.
  • Added a system setting under Appearance to hide the Countdown UI.
  • Resolved an issue where editing homebrew Weapon/Armor features showed Downtime Moves as the title. This now correctly shows Armor Feature/Weapon Feature.
  • Resolved an issue where the Delete button on Subclass items in a Class wasn't working when the link was broken.
Content Modules
  • The Compendium Browser is now checking on certain links it wasn't before. This makes broken links cause it to error out.

Frequently Asked Questions

Why is it called Foundryborne?

Communities in Daggerheart are all named -borne (Wildborne, Seabornde, Loreborne... etc.). Since this is a Community Project, we decided to name it Foundryborne.

(How) Can I contribute?

You can find a link to the repository in the menu on this website. Please fork the repo & create a pull request. We also recommend to join the Discord server.

Check the Milestones and grab an unassigned task.

What about translations?

Currently, community translations should be added as a module (there are already a few available!). For the officially announced translations we want to wait for their release first.

Which FoundryVTT version will I need for this system?

We are developing for v13. If you don't want to upgrade yet for any reason, you can use the portable version of v13 temporarily until you are ready.

What features/automations will there be?

We strife to automate as much as possible - where it makes sense. It will also be possible to disable automations in the system settings. You will also find a lot of settings for supporting homebrew content. We aim to include the complete SRD content as well.

There will be character creation automation, as well as levelling up automation. We have a global fear tracker that can be customized, a robust countdown system, and even support for the optional action tracker, where players can request the spotlight from the GM and have 3 action tokens each.
This list is not complete.

What about the Void playtest content?

The Void playtest materials are not part of the SRD and therefore can't be included with the system. However, the system makes it easily possible to add custom domains and (sub)classes.

Will you include the official artwork?

The official artwork is not part of the SRD and therefore can't be included with the system. However, you will be able to provide your own artwork for everything. We aim to include as much (non-official) artwork as possible.

What about Demiplane integration?

Demiplane currently has no API to make any integration possible. However, we have reached out to them and will be actively talking with the Demiplane to see if any kind of integration would be possible - or makes sense. This is most likely a feature for a later release however, or possibly even a separate module.

I have another Question!

Please join the Discord Server, especially the #support channel. If your question has not been answered, please use the ticket system in #contact-us.
Please do not DM the dev team directly, as we are getting too many messages which are distracting us from our work.
